# Agent Architecture

The AI Chief of Staff acts as an autonomous loop engine that continuously processes a task queue, manages persistent state, and operates a Google Drive folder as its workspace.

## Core Components

- **DriveStateManager**: Reads and writes the agent's state directly to Google Docs inside the Drive workspace folder. There are no local state files. The three state documents are:
    - `todo`: The current pending task queue (JSON array).
    - `memories`: Abstract persistent key-value knowledge store (JSON object).
    - `skills`: Learned procedures and macros (JSON object).
- **AIAgent**: The core engine that uses JSON-mode outputs from local Ollama models. Loads full context from the Drive state on every iteration and dispatches actions based on the model's response.
- **Google Drive Workspace**: The agent has full read/write/delete control over its assigned Drive folder. State documents, the manager chat log, and any files the agent creates all live here.
- **Manager Chat**: A Google Doc named `Manager Chat` inside the workspace folder. The agent appends `[AI]:` messages when it needs input; the manager writes replies directly into the doc.
- **Code Executor** (`code_executor.py`): Runs agent-authored Python snippets in a subprocess using the project's venv interpreter. `PROJECT_DIR` is prepended to `PYTHONPATH` so project modules are importable. Captures stdout/stderr and enforces a 30-second timeout.
- **drive_bot_tools** (`drive_bot_tools.py`): Helper module available inside executed code. Provides pre-authenticated, config-aware wrappers for all Google APIs the agent has access to:
    - `search(query, num)` — Google Custom Search, returns `[{title, link, snippet}]`
    - `list_files()` — lists Drive workspace files
    - `read_file(name)` — reads a file's text content
    - `write_file(name, content)` — creates or overwrites a file
    - `delete_file(name)` — deletes a file

## Local Files

Only two files are stored locally:

| File | Purpose |
|---|---|
| `config.json` | Bootstrap config: model name, `folder_id`, `chat_doc_id`, `search_api_key`, `search_cx` |
| `oauth_credentials.json` | OAuth2 Desktop app client secrets (copied from user-supplied path at setup) |
| `token.json` | OAuth2 access + refresh token (generated by the browser auth flow at setup) |

## Actions

The agent responds with a single JSON object per iteration. Valid actions:

- `think`: Processes intermediate reasoning without side effects.
- `add_memory`: Writes a key-value pair to the `memories` doc in Drive.
- `update_skill`: Writes a named procedure to the `skills` doc in Drive.
- `add_task`: Appends a new task to the `todo` doc in Drive.
- `complete_task`: Removes a completed task from `todo` and appends a result summary to `Manager Chat`.
- `ask_manager`: Appends an `[AI]:` question to `Manager Chat` and polls every 10 seconds until the manager writes a reply. The reply is fed into the next iteration as `doc_input`.
- `list_files`: Lists all files currently in the Drive workspace folder. The result is fed back into the next iteration.
- `create_file`: Creates a new Google Doc in the workspace folder with optional initial content.
- `read_file`: Reads the text content of a named file from the workspace. The content is fed back into the next iteration.
- `write_file`: Overwrites the content of a named file in the workspace (creates it if it does not exist).
- `delete_file`: Permanently deletes a named file from the workspace folder.
- `run_code`: Executes a Python snippet via `code_executor.run_code()`. The snippet runs in a sandboxed subprocess with `drive_bot_tools` importable. Stdout is captured and fed back into the next iteration as `code_result`.

## Data Flow

```
main.py : run_loop()
  └─> agent.run_step(input_text)
       ├─> DriveStateManager  [reads todo, memories, skills from Drive]
       ├─> ollama_client.chat()  [sends system prompt + context to local LLM]
       ├─> parse_response()  [extracts JSON action from model output]
       └─> dispatch action
            ├─> state actions  →  DriveStateManager._save()
            ├─> ask_manager    →  append_to_doc() + _wait_for_manager()
            │                       returns ("manager_replied", reply)
            ├─> list_files     →  list_files_in_folder()
            │                       returns ("files_listed", summary)
            ├─> read_file      →  read_doc_text()
            │                       returns ("file_read", content)
            ├─> create_file    →  get_or_create_doc() + overwrite_doc()
            ├─> write_file     →  get_or_create_doc() + overwrite_doc()
            ├─> delete_file    →  find_file_in_folder() + delete_file_by_id()
            └─> run_code       →  code_executor.run_code()
                                      subprocess with drive_bot_tools available
                                      returns ("code_result", stdout+stderr)

  "manager_replied" | "files_listed" | "file_read" | "code_result"
       └─> input_text set for next iteration
```
